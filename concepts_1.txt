const is used to declare something constant (read-only). Once a value is declared const,
 it cannot be modified after initialization. It helps make programs safer and prevents accidental changes.



 A const variable must be initialized when declared and cannot change later.

A CONST MEMBER FUNCTION promises not to modify the object’s data members. Matlab ki wo function ko agar call bhi karoge so it will not change any of your data members like health,level etc.
Basically:
You cannot modify data members
You cannot call non-const functions(bc they have tendency to change data members value)

CONST OBJECT:
You cannot change any data members of a const object
You cannot call any non const function (bc they have tendency to change data members value)


CONST DATA MEMBER: a data member which cannot be changed after initialization


CONST WITH FUNCTION PARAMETERS:
Basically these parameter's value cannot be changed inside the function

CONST WITH POINTERS:

1.) Pointer to const data
(const int* p)
basically point to const of some datatype
you cannot change value of data type using pointer

2.) Const pointer
(int *const p)
basically pointer jis address ko point kar raha that cannot be changed
but you can change the value @ that address

3.) const pointer to const data
const int * const p;



Now the question arrises that atleast once you would like to initialise the const values as well ... how do we do that??

#INITIALISATION LIST

const variables must be initialised, not assigned

In C++ OOP, a constructor initialization list is a way to initialize class data members before the constructor body executes. It is written after the constructor name and before the constructor body.

General Syntax:
Constructor(parameters) : member1(value1), member2(value2)
{
    // constructor body
}

Eg: Circle() : pi(3.14) { }

1.) For const variables(explained above)
2.) For reference in classes
3.)For Base Class Constructors (Inheritance)
4.)For Member Objects


2.)-> In C++ int &x; gives an error
        you have to assign a value to reference variable
        Similarly, this is done in classes

        Eg:
        class Test {
        int &ref;

        public:
        Test(int &x) : ref(x) { }
        };


Important rule:
Members are initialized in the order they are declared in the class, NOT in the order of the initialization list.
