##VIRTUAL FUNCTION: Iss ka matlab hai ki we want to achieve run time POLYMORPHISM

Ab we want that if a base class pointer points to a derived class object, the derived class object's function should be used, this is done with help of virtual function

kuchh ni krna bas base class k display function k aage virtual likh do :)

class base{
    public : int x;
    virtual void display(){
        cout<<"You're in base class and the value of x is "<<x<<"\n";
    }
}
class derived : Public base{
    public : 
    int y; 
    void display(){
        void display(){
        cout<<"You're in derived class and the value of x is "<<x<<" and the value of y is "<<y<<"\n";
    }
    }
}


virtual keyword helps to run functions of the derived class

#NOTE : If a class has any virtual functions, its destructor should also be declared virtual.

#very important : Virtual destructors 
If your base class has a virtual function then you customly have to write a virtual destructor,too
Kyunki The pointer will be of base class and the default destructor will only delete things of base class 
To delete the derived class elements -
    virtual ~Base() {}
OTHERWISE : There can be memory leaks


NOTE: Run time POLYMORPHISM : late binding
      Compile Time POLYMORPHISM : early binding






A Good example:


#include <iostream>
using namespace std;

class Employee
{
  public:
    virtual void raiseSalary()
    {
        cout << "Employee salary raised (general)" << endl;
    }

    virtual void promote()
    {
        cout << "Employee promoted (general)" << endl;
    }

    // virtual destructor
    virtual ~Employee()
    {
    }
};

// Derived class: Manager
class Manager : public Employee
{
  public:
    void raiseSalary() override
    {
        cout << "Manager salary raised with incentives" << endl;
    }

    void promote() override
    {
        cout << "Manager promoted to Senior Manager" << endl;
    }
};

// Derived class: Engineer
class Engineer : public Employee
{
  public:
    void raiseSalary() override
    {
        cout << "Engineer salary raised with bonus" << endl;
    }

    void promote() override
    {
        cout << "Engineer promoted to Senior Engineer" << endl;
    }
};

int main()
{

    // Create different employees
    Manager m;
    Engineer e;

    // Array of base class pointers
    Employee *employees[2] = {&m, &e};

    // Raise salary for all employees
    cout << "--- Raising Salaries ---" << endl;
    for (int i = 0; i < 2; i++)
    {
        employees[i]->raiseSalary();
    }

    // Promote all employees
    cout << "\n--- Promotions ---" << endl;
    for (int i = 0; i < 2; i++)
    {
        employees[i]->promote();
    }

    return 0;
}



Output
--- Raising Salaries ---
Manager salary raised with incentives
Engineer salary raised with bonus

--- Promotions ---
Manager promoted to Senior Manager
Engineer promoted to Senior Engineer




##OBJECT SLICING:

What if  instead of assigning a bas_class_pointer = &derived_class_object
I do
base_class_object=derived_class_object
will i still be able to use derived class functions declared as virtual in base class ?

No — in that case you will NOT get polymorphic behavior. What you’re describing leads to something called object slicing.


Even if your display function is virtual, the object you have made is in itself a base class obj , toh ye toh hona hi tha

We can avoid the above-unexpected behavior with the use of pointers or references. Object slicing doesn't occur when pointers or references to objects are passed as function arguments since a pointer or reference of any type takes the same amount of memory.

Member Function of Another Class as Friend Function: Here Forward declaration of the base class(or any other class whose Friend we want to make this class)

Define it later after the base class


#include <iostream>
using namespace std;

// Forward Declaration needed
class base;


class GFG {
public:
    void GFG_Function(base& obj);
};


class base {
private:
    int private_variable;

protected:
    int protected_variable;

public:
    base() {
        private_variable = 10;
        protected_variable = 99;
    }


    friend void GFG::GFG_Function(base&);
};


void GFG::GFG_Function(base& obj) {
    cout << "Private Variable: " << 
             obj.private_variable
         << endl;
    cout << "Protected Variable: " << 
             obj.protected_variable;
}

int main() {
    base object1;
    GFG object2;
    object2.GFG_Function(object1);

    return 0;
}
